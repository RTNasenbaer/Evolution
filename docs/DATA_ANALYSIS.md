# Data Analysis Guide

## Overview

The Evolution Simulation provides comprehensive data export capabilities for analyzing entity behavior, population dynamics, and trait evolution over time. This guide covers CSV export, batch simulation, and data analysis techniques.

## CSVExporter Utility

The simulation uses a centralized `CSVExporter` class (`src/export/CSVExporter.java`) that provides consistent CSV formatting across all interfaces (Terminal, GUI, and Batch modes).

**Key Features:**

- Automatic header generation
- Append mode for continuous data collection
- Timestamped filename generation
- File writability validation
- Consistent formatting across all export types

**Supported Export Methods:**

1. `exportBiomeCounts()` - Simple population counts per biome
2. `exportEntityDetails()` - Detailed per-entity data with traits
3. `exportBiomeDetails()` - Ecosystem-level statistics per biome

**Usage in Code:**

```java
import export.CSVExporter;

// Export entity details
CSVExporter.exportEntityDetails(world, stepNumber, "entity_data.csv");

// Export biome details
CSVExporter.exportBiomeDetails(world, stepNumber, "biome_data.csv");

// Create timestamped filename
String filename = CSVExporter.createTimestampedFilename("results", "csv");
```

## CSV Export Formats

### Current Export: Population Data (Biome-Based)

**File Format**: `batch_results_TIMESTAMP.csv` or `gui_batch_results_TIMESTAMP.csv`

**Structure**:

```csv
Step,GRASSLAND,FOREST,WATER,DESERT,MOUNTAIN,SNOW
0,15,12,0,8,5,3
1,16,13,0,7,5,4
2,17,14,0,8,6,3
...
```

**Columns**:

- `Step`: Simulation step number
- Biome columns: Entity count in each biome type

**Generated By**:

- Terminal mode: `/export` command
- GUI mode: Batch simulation feature

### Entity-Level Data Export (Trait Analysis)

**Status**: ✅ Implemented via CSVExporter

**Purpose**: "Analysis of the survival ability of organisms by varying specific traits under different spatial conditions using cellular automata"

**Implementation**: Uses `CSVExporter.exportEntityDetails()` for consistent formatting across all interfaces.

**File Format**: `entity_details_TIMESTAMP.csv`

**Usage (Terminal Mode)**:

```
> /export details
✓ Entity details export ENABLED. File: entity_details_1234567890.csv

> /run 100
▶ Running 100 simulation step(s)...
✓ Completed 100 step(s). Current step: 100

> /export details
✓ Entity details export DISABLED. Data saved to: entity_details_1234567890.csv
```

**Structure**:

```csv
Step,EntityID,X,Y,Energy,Age,Speed,Mass,EnergyEfficiency,ReproductionThreshold,SightRange,MetabolismRate,MaxLifespan,BiomeType,HasFood
0,E001,45,67,100.0,0,1.2,80.5,0.95,85.0,5.5,1.05,950,GRASSLAND,true
0,E002,12,34,100.0,0,0.8,95.2,1.15,75.0,6.2,0.92,1050,FOREST,false
1,E001,46,67,95.3,1,1.2,80.5,0.95,85.0,5.5,1.05,950,GRASSLAND,false
...
```

**Columns**:

- **Step**: Simulation step number
- **EntityID**: Unique entity identifier
- **Position**: X, Y coordinates
- **Energy**: Current energy level
- **Age**: Steps survived
- **Genetic Traits**:
  - Speed: Movement rate
  - Mass: Entity mass
  - EnergyEfficiency: Energy use multiplier
  - ReproductionThreshold: Energy needed to reproduce
  - SightRange: Food detection distance
  - MetabolismRate: Base energy consumption
  - MaxLifespan: Maximum age
- **Environment**:
  - BiomeType: Current tile biome
  - HasFood: Food availability on current tile

### Biome-Level Data Export (Ecosystem Analysis)

**Status**: ✅ Implemented via CSVExporter

**Purpose**: Analyze biome-level statistics including food availability and entity distribution per biome type.

**Implementation**: Uses `CSVExporter.exportBiomeDetails()` for consistent formatting across all interfaces.

**File Format**: `biome_details_TIMESTAMP.csv`

**Usage (Terminal Mode)**:

```
> /export biomes
✓ Biome details export ENABLED. File: biome_details_1234567890.csv

> /run 100
▶ Running 100 simulation step(s)...
✓ Completed 100 step(s). Current step: 100

> /export biomes
✓ Biome details export DISABLED. Data saved to: biome_details_1234567890.csv
```

**Usage (GUI Mode)**:

- Click "Export Biome Details" button to start/stop export
- Run simulation normally
- Data is saved to file automatically each step

**Structure**:

```csv
Step,BiomeType,TotalTiles,TilesWithFood,EntityCount,AvgEntityEnergy,AvgEntityAge,EntityIDs
0,GRASS,850,234,12,95.30,5.2,"E000;E001;E002;E003;E004;E005;E006;E007;E008;E009;E010;E011"
0,FOREST,420,156,5,102.45,3.8,"E000;E001;E002;E003;E004"
0,WATER,180,0,0,0.00,0.0,""
0,DESERT,320,45,3,88.20,7.1,"E000;E001;E002"
0,MOUNTAIN,540,89,8,91.15,4.5,"E000;E001;E002;E003;E004;E005;E006;E007"
0,SNOW,190,12,1,85.50,2.0,"E000"
```

**Columns**:

- **Step**: Simulation step number
- **BiomeType**: Type of biome (GRASS, FOREST, WATER, DESERT, MOUNTAIN, SNOW)
- **TotalTiles**: Number of tiles of this biome type in world
- **TilesWithFood**: Number of tiles with food available
- **EntityCount**: Number of entities currently in this biome
- **AvgEntityEnergy**: Average energy of entities in this biome
- **AvgEntityAge**: Average age of entities in this biome
- **EntityIDs**: Semicolon-separated list of entity IDs in this biome

**Use Cases**:

- Compare biome habitability (entity count vs food availability)
- Identify which biomes support larger populations
- Track entity migration patterns between biomes
- Analyze food scarcity impact on population distribution
- Study biome-specific survival rates

## Experimental Designs

### Experiment 1: Trait Selection Pressure

**Hypothesis**: High-speed entities will dominate in sparse food environments

**Setup**:

1. Generate world with low food availability (desert-heavy)
2. Initialize 50 entities with varied speeds
3. Run 2000 steps
4. Export entity-level data

**Analysis**:

```python
# Compare initial vs final speed distribution
initial = data[data['Step'] == 0]
final = data[data['Step'] == 2000]

fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 5))
ax1.hist(initial['Speed'], bins=20)
ax1.set_title('Initial Speed Distribution')
ax2.hist(final['Speed'], bins=20)
ax2.set_title('Final Speed Distribution')
```

### Experiment 2: Biome Adaptation

**Hypothesis**: Entities will evolve traits suited to their primary biome

**Setup**:

1. Generate world with distinct biome regions
2. Initialize entities in each biome
3. Run 5000 steps
4. Track trait evolution per biome

**Analysis**:

```python
# Compare trait distributions by biome
for trait in ['Speed', 'Mass', 'SightRange']:
    data.boxplot(column=trait, by='BiomeType')
    plt.title(f'{trait} Distribution by Biome')
```

### Experiment 3: Population Stability

**Hypothesis**: Certain world seeds lead to stable populations

**Setup**:

1. Test 100 different seeds
2. Run each for 3000 steps
3. Measure population variance

**Analysis**:

```python
# Calculate coefficient of variation for each seed
results = []

for seed in seed_list:
    data = pd.read_csv(f'results_{seed}.csv')
    total_pop = data[biome_columns].sum(axis=1)
    cv = total_pop.std() / total_pop.mean()
    results.append({'seed': seed, 'cv': cv})

results_df = pd.DataFrame(results)
results_df.sort_values('cv').head(10)  # Most stable seeds
```

## Statistical Analysis

### Descriptive Statistics


### Hypothesis Testing

**Example: Does speed affect survival?**

```python
from scipy import stats

# Split entities into fast/slow groups
median_speed = entity_data['Speed'].median()
fast = entity_data[entity_data['Speed'] > median_speed]['Age']
slow = entity_data[entity_data['Speed'] <= median_speed]['Age']

# T-test
t_stat, p_value = stats.ttest_ind(fast, slow)
print(f'T-statistic: {t_stat}, P-value: {p_value}')
```

### Regression Analysis

```python
from sklearn.linear_model import LinearRegression

# Predict lifespan from traits
X = entity_data[['Speed', 'Mass', 'EnergyEfficiency', 'SightRange']]
y = entity_data['Age']

model = LinearRegression()
model.fit(X, y)

print('Trait Coefficients:')
for trait, coef in zip(X.columns, model.coef_):
    print(f'  {trait}: {coef:.4f}')
```

## Performance Considerations

### Large Dataset Handling

**Problem**: 50 entities × 5000 steps = 250,000 rows

**Solutions**:

1. **Sampling**: Export every 10th step
2. **Compression**: GZIP compress CSV files
3. **Columnar Formats**: Use Parquet for large datasets
4. **Database**: SQLite for complex queries

## Troubleshooting

### Large File Sizes

- Use sampling (every N steps)
- Enable compression
- Consider binary formats (Parquet, HDF5)

### Missing Data

- Check export configuration
- Verify simulation completed
- Ensure no crashes during export

### Performance Issues

- Process data in chunks
- Use efficient data structures (NumPy arrays)
- Parallel processing for independent analyses

## Simulation Balance

The simulation has been carefully tuned to produce meaningful data for trait analysis. Understanding the balance is crucial for interpreting results:

- **Energy Systems**: See [BALANCE_ADJUSTMENTS.md](BALANCE_ADJUSTMENTS.md) for detailed energy costs, food values, and metabolism rates
- **Trait Ranges**: Understand viable trait values and mutation ranges - each trait has natural limits
- **Biome Characteristics**: Each biome has distinct food availability and environmental stress levels

**Key Balance Points**:

- Entities start with 50 energy (sufficient for exploration and reproduction)
- Food energy ranges from 1-10 depending on biome quality
- Movement costs scale with speed, mass, and biome difficulty
- Reproduction threshold of 40 energy is achievable within typical lifespan
